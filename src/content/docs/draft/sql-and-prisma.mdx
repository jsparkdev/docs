---
title: SQL and Prisma
---

SQLite와 같은 데이터베이스를 쿼리하기 위해서는 SQL 사용. SQL은 DB와 통신할 때 사용되는 언어

SQL의 기본은 테이블, 테이블은 행과 열의 모음, 행은 엔티티, 열은 엔티티의 속성

Prisma는 JS앱에서 DB와 쉽게 통신하기 위한 도구의 모음. Prisma의 기능 중 하나가 ORM. 이는 DB와 통신하기 위해 JS 함수 및 객체를 사용함.

그리고 Prisma는 스키마를 정의하는 언어임. 스키마는 SQL에 비해 읽기도, 관리하기도 쉬움. 스키마를 통해 테이블 간 관계를 정의, 각 열의 타입을 정의함.

스키마를 SQL로 변환하고 실행하여 테이블을 생성함. 이는 npx prisma db push로 .

거꾸로 이미 데이터베이스가 존재하고, 그걸 이용해 스키마를 만들어야 한다면 npx prisma db pull.

특히, push 이후 자동으로 제너레이터가 실행되므로 별도의 npx prisma generate를 실행할 필요는 없음.

스튜디오를 이용해 데이터를 추가할 수도 있음. 데이터 추가 후 npx prisma db push.
