---
title: Prisma Migration
---

1. `prisma db push` : 마이그레이션 파일 생성 없이 스키마를 바로 DB에 적용합니다.
2. `prisma db pull` : 연결된 데이터베이스로부터 스키마를 추출하여 파일에 복사합니다.
3. `prisma migrate dev` : 스키마 파일로부터 마이그레이션 파일을 생성한 후, 이를 연결된 데이터베이스에 적용합니다.
4. `prisma migrate reset` : DB에 적용된 모든 마이그레이션을 제거한 후, 모든 마이그레이션을 처음부터 다시 적용합니다. 이후 시드 스크립트를 실행하여 필수 데이터를 초기화시킵니다.
5. `prisma migrate deploy` : 존재하는 모든 마이그레이션 파일들을 연결된 데이터베이스에 적용합니다. 배포 프로세스에서 작동하도록 설정합니다.

## 워크플로

1. 개발 모드에서 `prisma migrate dev` 및 `prisma migrate reset`을 사용하여 최적의 마이그레이션을 찾습니다.
2. 최적의 마이그레이션을 찾았다면 데이터베이스를 초기화시키고 모든 마이그레이션을 처음부터 다시 적용하기 위해 `prisma migrate reset`을 사용합니다.
3. 사이트 빌드 및 배포 프로세스에 `prisma migrate deploy`를 사용하도록 설정하여 데이터베이스에 모든 마이그레이션을 적용화여 최신 스키마를 유지합니다.
