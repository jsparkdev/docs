---
title: Prisma Migration
---

1. `prisma db push` : 마이그레이션 파일 생성 없이 스키마를 바로 DB에 적용합니다.
2. `prisma db pull` : 연결된 데이터베이스로부터 스키마를 추출하여 파일에 복사합니다.
3. `prisma migrate dev` : 스키마 파일로부터 마이그레이션 파일을 생성한 후, 이를 연결된 데이터베이스에 적용합니다.
4. `prisma migrate reset` : DB에 적용된 모든 마이그레이션을 제거한 후, 모든 마이그레이션을 처음부터 다시 적용합니다. 이후 시드 스크립트를 실행하여 필수 데이터를 초기화시킵니다.
5. `prisma migrate deploy` : 존재하는 모든 마이그레이션 파일들을 연결된 데이터베이스에 적용합니다. 주로 배포 파이프라인에 사용되므로 비대화형입니다.
