title: Migration
---

없는 필드를 그냥 추가하는건 쉬움. 그냥 추가하고 push하면 됨.

근데 기존 데이터에 영향을 주는 작업 (선택 사항이었던걸 필수로 만들거나 중복 가능 데이터를 유니크하게 만드는)은 push가 안될거임.

왜냐면 이미 null인데 필수가 되거나, 중복된 데이터가 있는데 유니크로 만들려고 하니까.

이 때 필요한게 마이그레이션

프리즈마는 마이그레이션 명령을 하면 sql 파일을 생성해 저장함. 이 파일은 변경할 수도 있음.

그리고 이파일은 git에 배포되어야 함. 그래야 시간에 따른 디비 변경사항을 추적할 수 있음.

그냥 앱을 배포할때 npx prisma migrate deploy 만 하면 프리즈마가 알아서 다 해줌.

push 랑 migrate가 뭐가 다른데? 전자는 내 스키마를 실험하는 용도임. 후자는 디비에 커밋하는 용도임.

마이그레이션 실제로 해보자 ㄱㄱ. npx prisma migrate dev

이러면 prisma/migrations 폴더에 sql 파일이 포함된 폴더가 생성됨.

그리고 실제 배포시엔 npx prisma migrate deploy 사용.
