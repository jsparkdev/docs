---
title: Seed
---

앱 테스트용 데이터를 디비에 추가하거나 필요한 데이터로 디비를 초기화하는 것.

개발에서의 시딩?
물론 스튜디오에서 하나씩 추가할 수 있지만, 실수할 수 있고, 많은 데이터를 작성하기 힘들기 때문에 디비를 리셋하는 스크립트 사용.
이 스크립트는 멱등성을 갖추는게 좋음. 즉, 몇 번을 실행해도 원하는 결과를 얻어야 함. 이를 위해 스크립트는 디비의 모든 데이터를 삭제한 후, 데이터를 추가해야 함.

프로덕션에서의 시딩?
가장 좋은 방법으 마이그레이션 스크립트에 포함하는거임. 즉, 프리즈마가 생성하는 migration sql 파일을 수정하여 필수 데이터를 추가할 수 있음.

프리즈마의 시딩?
시드 스크립트를 작성하면 npx prisma migrate reset 실행 후 시드 스크립트가 자동으로 실행 됨.
물론 npx prisma db seed로 시딩을 따로 수행할 수도 있음.

가장 먼저 해야할 일은 패키지에 코드 추가

```json
"prisma": {
	"seed": "npx tsx prisma/seed.ts"
}
```

정리하자면, 이거 추가하고 npx prisma migrate reset 실행하면 마이그레이션 + 시딩이 실행됨.
