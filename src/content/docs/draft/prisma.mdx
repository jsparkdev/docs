스키마 -> DB : npx prisma db push
DB -> 스키마 : npx prisma db pull

파괴적인 작업 - 마이그레이션 필요
배포시 - npx prisma migrate deploy (변경사항을 포로덕션 DB에 적용, 로컬에서 실행 X)
개발에서 마이그레이션 생성, 적용 - npx prisma migrate dev (개발환경 전용)
개발에서 특정 작업, push 취소 - npx prisma migrate reset (개발환경 전용)

npx prisma migrate reset: 시드 스크립트 실행
npx prisma migrate dev, deploy : 시드 스크립트 실행 X - npx prisma db seed 필요
