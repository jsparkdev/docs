디원 리믹스 테스트

디비 생성
npx wrangler d1 create 디비이름

바인딩 구성 복사해서 toml 파일로 붙여넣기
이게 바인딩임. 워커가 d1에 액세스가능해짐.

스키마 sql 파일 만들어서 디비 초기화

일단 이걸 로컬에서 초기화시키기 (스키마 sql파일 실행)
npx wrangler d1 execute 이름 --local --file=스키마파일경로
이걸로 초기화하면 로컬에 디비가 생성되어 그 디비가 초기화됨

초기화된 로컬 디비를 쿼리해보자.
npx wrangler d1 execute 이름 --local --command="SELECT * FROM Customers"
그러면 콘솔에 쿼리 결과가 출력됨.

이렇게 로컬에서 테스트 완료함. 이제 D1에서 쿼리를 해야함.
스크립트를 실행해서 이 작업을 수행해야 함.

(내가 임의로 추가한 단계)
npx wrangler types : 디비의 타입 생성

이제 로더에서 데이터를 가져오자. context 객체에서 이제 DB에 접근 가능.
이 DB의 prepare 메서드의 인자로 SQL쿼리를 넣어주자.
그리고 bind로 플레이스홀더에 값 바인딩, all로 쿼리 실행 후 결과 반환하도록 하고
그 결과를 리턴하여 UI에서 사용할 수 있도록 하자.

개발 모드에서 결과가 원하느대로 나오면 이제 D1에 결과를 적용해야함.
이 작업은 로컬 디비에 하던 작업에 --local 만 제거하면 됨.
npx wrangler d1 execute 이름 --local --file=스키마파일경로 (이걸로 디비 초기화)
npx wrangler d1 execute 이름 --local --command="SELECT * FROM Customers" 이걸로 D1의 쿼리 테스트.

이제 웹사이트를 배포하면 되는데 그 전에 D1을 바인딩해야함.
Workers 및 Pages - 프로젝트 선택 - 설정 - 함수 - D1 데이터베이스 바인딩 추가

이후 배포하면 끝.
