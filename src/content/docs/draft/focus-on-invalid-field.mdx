title: Focus on Invalid Field in React
---

에러난 필드에 포커스 주기

우선 ref로 폼을찍고, 그 폼에 tabIndex -1로 주셈
탭인덱스 주는 이유는 폼 에러시 폼에 포커스주기위함.
(필드에러때 필드에 포커스주듯이)

일단 핵심은 인풋을 제출했을때 실패하면 에러객체, 성공하면 리디렉션임.
즉, 제출할 때마다 오류가 달라지므로 받아오는 액션데이터가 달라진다는거고
이게 바뀔때마다 다른 작업을 수행하기 위해 유즈이팩트를 사용
즉 디펜던시 어레이는 액션데이터가 되지.

```tsx
useEffect(() => {
  const formEl = formRef.current
  if (!formEl) return
  if (actionData?.status !== 'error') return

  if (formEl.matches('[aria-invalid="true"]')) {
    formEl.focus()
  } else {
    const firstInvalid = formEl.querySelector('[aria-invalid="true"]')
    if (firstInvalid instanceof HTMLElement) {
      firstInvalid.focus()
    }
  }
}, [actionData])
```

폼 없거나 액션데이터가 에러가 아니면 컷
만약 폼 그 자체에서 오류가 있으면 거기 포커스
없으면 쿼리실렉터로 하위 필드 한번 찾아보고, 그애가 HTML요소면 포커스

이렇게 복잡한 로직은 훅으로 빼는게 좋음.

일단 생각해보면 html 요소에 접근해야하므로 ref 써야하고, 완성된 form에 접근해야하므로 useEffect 쓰는게 맞음.
