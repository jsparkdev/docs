---
title: Remix Data Fetching Event
---

클라이언트 사이드 애플리케이션에 익숙한 경우 이벤트 핸들러에 데이터 페칭을 추가하는 경우가 많음.

근데 이건 중복 요청, 요청 중 페이지 이동, 실패 처리, 경쟁 조건 등 직접 해결해야 하는 문제가 많음.

이 문제를 해결하기 위해서는 버튼의 이벤트 핸들러에서 요청을 처리하는 대신 버튼을 `Form` 컴포넌트에 포함시켜 리믹스가 이걸 다 처리하도록 위임하여 모든 엣지케이스를 처리하도록 만드는게 좋음.

근데 가끔 여러 버튼의 POST 요청을 처리해야 하는 경우가 있음. 근데 action 함수는 하나뿐.

이 때는 각 버튼에 같은 값을 가진 `name`을 추가하고, 서로 다른 `value` 값을 추가합니다. (라디오버튼 생각)

그러면 action 함수에서 formData.get(name) 을 사용하면 클릭한 버튼의 value가 반환됩니다.

이를 이용해 분기처리 하면 됩니다.
