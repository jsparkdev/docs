패키지 매니저로 Bun을 사용하는 여러 패키지를 관리하는 경우, Bun 워크스페이스를 사용하여 패키지 관리를 효율적으로 할 수 있습니다.

그리고, Changeset을 사용하여 여러 패키지의 버전 관리를 효율적으로 할 수 있습니다.

## 사용법

<Steps>
1. Changeset을 설치합니다.

    ```bash
    bun add -D @changesets/cli --workspace-root
    ```

2. Changeset을 초기화합니다.

    ```bash
    bun changeset init
    ```

3. Changeset의 구성 파일에서 `access` 필드를 `public`으로 설정합니다.

    ```json
    {
      "access": "public"
    }
    ```

4. 각 패키지의 코드를 업데이트합니다.

5. 코드를 모두 업데이트했다면 다음 명령을 사용하여 Changeset을 생성합니다.

    모든 응답에 답변하면 변경 사항이 기록된 Markdown 파일이 생성됩니다.

    ```bash
    bun changeset add
    ```

    변경 사항이 많아 여러번 저장해야 한다면 이 명령을 여러번 실행해야 합니다.

6. 모든 변경사항과 생성된 Markdown 파일을 커밋합니다.

    ```bash
    git add .
    git commit -m "feat: update packages"
    ```

7. 변경 사항 커밋을 생성하였으므로, 이제 버전 업데이트 커밋도 필요합니다.

    우선 다음 명령을 사용하여 만들어진 Markdown 파일을 이용해 실제 버전을 업데이트합니다.
  
    ```bash
    bun changeset version
    ```

8. 버전 업데이트를 커밋합니다.

    ```bash
    git add .
    git commit -m "chore: update version"
    ```

9. 모든 변경 사항이 적용되고, 버전도 업데이트 되었으므로 NPM에 배포합니다.

    ```bash
    bun changeset publish
    ```

10. 지금까지 수행한 모든 커밋을 푸시합니다.

    ```bash
    git push
    ```
</Steps>
