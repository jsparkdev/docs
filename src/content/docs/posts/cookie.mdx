---
title: Cookies
---

- 특정 기간 동안 유지되는 웹 애플리케이션의 상태 관리를 위해 사용됩니다.
- 브라우저에 저장되며, 요청 전송 시 함께 전송됩니다. 이 덕분에 서버는 세션의 정보, 사용자의 기본 설정 등을 기억할 수 있게 됩니다.
- 로컬 스토리지는 클라이언트 측 JavaScript에서 바로 접근이 가능하여 요청과 함께 전송되지 않아야 하는 대량의 데이터를 저장하는데 적합한 반면, 쿠키는 세션 관리 및 요청과 함께 전송되어야 하는 소량의 데이터를 저장하는데 적합합니다.
- 또한 로컬 스토리지는 요청에 포함되지도 않으므로, 서버가 초기 페이지를 렌더링할 때 사용자 정보 등의 정보를 사용할 수 없습니다. 그러므로 이를 활용하려면 서버에서 렌더링된 페이지가 클라이언트로 전송된 후, 로컬 스토리지의 값을 가져와 사용해야 하므로 성능이 좋지 않습니다.
- 쿠키는 클라이언트 측, 서버 측에서 모두 설정할 수 있지만, 서버에서 설정하는 것이 일반적입니다.
- 클라이언트 측에서 설정 시 `document.cookie` 를 사용할 수 있지만, `js-cookie`와 같은 라이브러리를 사용하는 것이 권장됩니다.
- 서버 측에서 설정 시 응답 객체의 헤더에 `set-cookie`로 값을 설정할 수 있지만, `cookie`와 같은 라이브러리를 사용하는 것이 권장됩니다.
- 브라우저는 기본적으로 응답을 받으면 `set-cookie` 헤더를 분석하여 쿠키의 값을 얻고, 이를 `cookie-jar`에 저장합니다.
- 이렇게 저장된 cookie는 브라우저가 후속 요청을 보낼 때 요청 객체의 헤더에 쿠키를 포함합니다.
- 쿠키는 `path` 속성을 통해 특정 경로 및 해당 경로의 하위 경로로 향하는 요청에만 쿠키를 포함시킬 수 있습니다.
