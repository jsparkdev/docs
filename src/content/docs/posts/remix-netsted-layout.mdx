---
title: Remix Nested Layout
---

기본적으로 URL 중첩이 일어나면 자식 컴포넌트는 부모 컴포넌트의 Outlet에 렌더링됩니다.

근데 URL만 중첩시키고 레이아웃은 중첩시키지 않고 싶을 때도 있음. 즉, URL은 하위에 있지만, 아예 별개의 페이지로 분류되고 싶을 때가 있음.

이 때는 \_ 를 사용함. \_를 부모 경로 뒤에 추가하면 이 부모의 자식들은 부모 컴포넌트의 Outlet에 렌더링되지 않고 별도의 페이지처럼 렌더링 됨.

즉, 부모 컴포넌트를 대체해야하는 경우 사용함. 이는 부모 컴포넌트에 Outlet을 사용할 필요가 없다는 것을 의미하기도 함.
