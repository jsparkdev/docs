---
title: Relation
---

User 모델과 SocialSecurityNumber 모델이 있다고 가정.

이 때, 한 명의 user는 한 개의 SSN을 가집니다. 이는 1대 1 관계이며, 소유 당하는 쪽인 SSN은 user 및 userId 필드를 가지게 됩니다.

그리고 이 때 SSN의 user 필드는 @relation을 통해 User 모델과 연결됩니다.

- fields : SSN의 어떤 필드가 연결될지 설정합니다.
- references : User 모델의 어떤 필드가 연결될지 설정합니다.

즉, fields가 userId이고, references가 id라면 SSN.userId는 User.id를 참조합니다. 이 때 userId 필드를 외래키라고 합니다.

특히, 1대 1 관계에서 userId는 유니크합니다. 왜냐면 여러 SSN이 존재하더라도 각 SSN은 오직 한 명의 서로 다른 User를 가리키기 때문입니다.

절대 여러 SSN이 같은 사람을 가리킬 수 없습니다.

그리고1대 1 관계에서 User는 SSN을 소유하지 않을 수도 있습니다. 즉, 프리즈마에서 1대 1은 필수 관계가 아닙니다.

## 일대 다 관계

User 모델은 여러 개의 Post 모델을 소유할 수 있으며 이게 바로 일대 다 관계입니다.

여러 Item이 한 명의 User의 소유일 수도 있기 때문에 userId는 유니크하지 않습니다.

## 다대 다 관계

그냥 서로 배열 필드를 가집니다. 참조를 위한 id나 레퍼런스가 필요하지 않습니다.

사실 관계형 데이터베이스에서는 두 테이블로 다대 다 관계를 만들 수 없습니다.

세 번째 테이블이 필요하지만, 프리즈마는 이를 만들어줍니다.

## 나누기

User, Item 모델이 있고, 각각 이미지 모델이 필요하다고 가정합니다.

이 때 Image 모델을 만들어 User.image, Item.image를 만들려고 할 수도 있습니다.

하지만 이는 좋은 선택이 아니며 UserIamge, ItemImage 모델을 따로 만드는 것이 좋습니다.

왜냐면 각각 다른 요구사항을 미래에 바로 추가할 수 잇기 때문입니다.
