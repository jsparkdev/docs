---
title: Redirect
---

- POST 요청 후 뒤로 가기 버튼을 누르면 양식 재제출 경고 알림이 나타납니다. 이는 브라우저가 동일한 요청을 다시 보내려고 하기 때문입니다.
- 즉, 이는 송금이나 물건 구매가 또 다시 일어날 수 있다는 의미입니다.
- 이러한 이유로 POST 요청에 대한 응답은 HTML이 아닌, 리디렉션을 사용해야 하며, 이를 위해 일반적으로 PRG 패턴이 사용됩니다.
- 리디렉션의 과정은 다음과 같습니다.
  1. POST 요청 전송
  2. 302 상태 코드와 함께 Location 헤더를 응답에 포함하여 클라이언트로 전송
  3. 클라이언트는 Location 헤더에 있는 주소로 GET 요청 전송
  4. 서버는 해당 주소에 해당하는 HTML을 응답으로 전송

리디렉션을 위한 코드는 다음과 같습니다.

```js
throw new Response(null, { status: 302, headers: { Location: '/login' } });
```

리믹스와 같은 프레임워크는 이를 단순화합니다.

```js
throw redirect('/login');
```
