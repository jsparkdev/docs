---
title: Redirect
---

POST 요청 후 뒤로 버튼을 누르면 양식 재제출 경고 알림이 뜹니다. 이는 브라우저가 요청을 다시 보내려고 하기 때문입니다. 즉, 뒤로 버튼을 눌렀을 때 송금이나 물건 구매가 또 다시 일어날 수 있다는 의미입니다.

이러한 이유로 POST 요청에 대한 응답은 HTML이 아닌, 리디렉션을 사용해야 합니다. 하지만, 리디렉션이 사용되면 페이지에서 보는 데이터는 최신 상태가 아닙니다. 왜냐면, 요청 보낸 후, 바로 페이지를 이동하기 때문입니다.

하지만 이러한 문제는 최신 프레임워크에서 Revalidation이라는 기술로 해결합니다.

리디렉션의 원리는 다음과 같습니다.

1. POST 요청 전송
2. 302 상태코드와 함께 Location 헤더를 응답에 포함하여 전송
3. 클라이언트는 Location 헤더에 있는 주소로 GET 요청을 보냄
4. 서버는 해당 주소의 UI를 응답으로 보냄

```js
throw new Response(null, { status: 302, headers: { Location: '/login' } });
```

리믹스와 같은 프레임워크는 이를 단순화합니다.

```js
throw redirect('/login');
```
