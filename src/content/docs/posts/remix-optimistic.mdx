---
title: Remix Optimistic UI
---

네트워크가 느리면 요청과 응답이 늦게 도달하므로 테마 변경도 즉시 이루어지지 않음. 이 때 필요한게 낙관적 UI, 네트워크 요청이 완료되지 않아도 해당 요청이 완료된 것 처럼 UI를 업데이트할 수 있음.

원리는 간단함. 요청을 보내고 보낸 요청의 데이터를 바로 UI에 반영하는 것임.

useFetchers 훅을 사용하면 현재 요청이 진행중이거나 방금 막 완료된 요청을 담은 fetcher 배열이 반환됨.

당연히 현재 진행중인 요청만이 formData를 가지지. 그 데이터를 가져와 UI를 업데이트하면 됨.

useFetchers를 사용하지 않고 테마 변경 fetcher에 key를 전달해서, 그 key를 가진 fetcher를 가져와 UI를 업데이트할 수도 있음.
