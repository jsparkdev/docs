GitHub actions

- 테스트, 빌드, 배포 파이프라인 자동화
- PR 빌드, 테스트나 병합된 PR을 배포
- 리포지토리에 이벤트 발생하면 워크플로우 실행 (예를들면 이슈달면 적절한 레이블 추가)
- 리눅스, 윈도우, 맥 가상머신 제공

- 핵심은 리포지토리에 이벤트 발생하면 실행되는 워크플로우 구성하는거.
- 워크플로우는 순차적이나 병렬로 실행될 수 있는 하나 이상의 Job(작업)을 포함
- 각 작업은 가상머신 러너나 컨테이너에서 실행
- 각 작업은 (action)액션을 실행하는 하나 이상의 step(단계)를 가짐

구조
- workflow: 전체 자동화 과정, 하나 이상의 job 포함, yaml, .github/workflows에 정의
- job: 특정 환경에서 실행되는 여러 step으로 구성, 기본적으로 종속성 없고 병렬로 실행, 종속성 있으면 순서대로 실행, 각 러너는 한번에 하나의 job만 실행 가능.
- step: 각 step에선 action 호출, 각 단계는 순서대로 실행됨. 
- action: 실제 실행 명령. 재사용 가능하도록 반복되는 코드를 패키징해놓은거.
